# MuleShield AI - Web Application

This is the Next.js frontend web application for the MuleShield AI platform.

## Directory Structure

Inside `src/` we have organized files following React/Next.js clean architecture patterns:
- `app/`: App router layouts and page components.
- `components/`: Global reusable presentation/UI components.
- `hooks/`: Custom stateful React hooks.
- `services/`: REST/GraphQL service integrations and API clients.
- `store/`: State management libraries configuration (e.g. Zustand, Redux).
- `context/`: React context providers.
- `types/`: Domain-specific TypeScript declarations.
- `utils/`: General utilities (formatting, date parsers, etc.).
- `constants/`: Global configurations, URLs, and static labels.
- `styles/`: Shared stylesheet files and global themes.
- `public/`: Public assets including logo images, icons, and fonts.
